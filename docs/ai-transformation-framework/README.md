## Purpose

This guide is the complete reference for USSP / Krasan's AI Transformation Framework -- a structured, evidence-based approach to augmenting SDLC processes with AI across SAFe and Scrum delivery teams. It covers every phase from initial governance through organization-wide scale, with practical tools, templates, and measurement methodology.

The framework is designed to be engagement-agnostic. It provides the methodology, processes, and instruments that USSP / Krasan teams use to deliver AI transformation for any client, in any industry, under any regulatory environment.

## Framework Phases

```
Govern --> Baseline --> Design --> Train --> Pilot --> Measure --> Playbook --> Scale
```

The framework follows eight phases, each building on the previous. It is designed to be executed sequentially for new engagements and can be entered at any phase for engagements with existing maturity.

## How to Use This Guide

1. **Before the engagement:** Read Ch 1-3 to understand the framework, then use Ch 2 (Executive Discovery) to run the initial client conversation.
2. **During setup:** Use Ch 4 (Assessment) to gauge readiness, Ch 5 (Baseline) to capture metrics.
3. **During delivery:** Follow the Design, Train, Pilot, Measure, Playbook, and Scale phases sequentially.
4. **For new engagements:** Use the Deployment chapter to set up everything from scratch.

## Philosophy

Three principles underpin every phase of this framework:

- **Amplify, Don't Replace** -- AI augments the existing team. Same people, better tools, more output. No workforce reduction.
- **Measure, Don't Guess** -- Every claim is backed by data. Baseline before AI, controlled pilot, research-backed measurement.
- **Pilot, Then Scale** -- Start with a small number of teams. Measure the impact. Scale what works.

## Methodology References

The framework draws on established research and industry standards:

- **DORA** (DevOps Research and Assessment) -- Deployment frequency, lead time, change failure rate, MTTR
- **SPACE** -- Satisfaction, Performance, Activity, Communication, Efficiency
- **QUS** (Quality User Story) -- 13-criteria story quality assessment
- **Six Sigma** -- Statistical rigor for before/after comparison
- **ISO/IEC 25010** -- Software quality characteristics
