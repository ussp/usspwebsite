Train

Everyone gets:

  • DoIT AI Policy compliance
  • Data guardrails & boundaries
  • Code generation guardrails (human-in-the-loop)

Role-specific (Krasan roles):

  • BA-Technical — use case quality, AC generation
  • Configuration — Dynamics config augmentation
  • Developers — code generation, review, docs
  • Testers — test script generation
  • Scrum Masters — sprint insights, metrics
Deliverable: Training Completion Report (per role)

Pilot

  • 2 teams — Krasan roles only initially
  • Remaining teams as control group
  • Sprints 3 & 4 of PI 20 (15% capacity for AI)
  • AI-assisted code gen from day 1 under HITL (stories, docs, tests, code — developer reviews/edits/accepts; peer PR approval)
  • Weekly check-ins to iterate
Deliverable: Weekly Check-in Reports

Measure

  • Tangible outputs — not story points
  • Lines of code generated & accepted
  • Documentation lines generated
  • Configuration lines generated
  • Cycle time, quality, rework reduction
  • Team health survey (before & after)
Target: 10-15% improvement in 13 weeks
