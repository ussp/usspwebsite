## Major Decisions & Shifts

### 1. AI Code Generation — BACK ON THE TABLE
- **Initial position:** Framework excluded AI code generation (Krasan's conservative approach)
- **Jim's response:** This constraint was "accidental" — he wants the team to reconsider
- **Decision:** Revisit code generation. Start with user story improvements, then introduce code generation in a stair-stepped approach
- **Action:** Consult Gartner for industry best practices and external validation
- **Impact:** This is a significant scope expansion from the original framework

### 2. Productivity Target — 10-15% in 13 Weeks
- **Jim set the target:** 10-15% productivity improvement within 13 weeks (1 PI)
- **Previous target was 5%** — Jim is more ambitious
- **Measurement:** Focus on tangible outputs (lines of code, documentation, configuration) — NOT story points
- **Impact:** Need to recalibrate KPI selection away from velocity/story points

### 3. Dave — Interim AI Lead During Jim's Absence
- **Jim designated Dave** as the primary contact for AI initiatives while Jim is unavailable
- **Jim will be gone for several weeks** after Tuesday
- **Communication:** Copy Jim on key updates via email
- **Impact:** All follow-up coordination goes through Dave

### 4. Governance — Gartner + State AI Policy
- **30-day notice required** for new AI deployments (confirmed by Jim)
- **Human-in-the-loop mandatory** for all AI-driven decisions
- **Agency is responsible** for risk acceptance and compliance
- **DoIT Core recommends** but agencies can select and procure additional tools if justified
- **Need governance team** — establish or leverage existing AI governance structure
- **Jim suggested Gartner** for external validation of approach

### 5. PMO and BA Teams — Approved for Pilot Involvement
- Jim approved engaging PMO and BA teams in the pilot
- **Caution:** Potential resistance — Jim offered executive support if needed
- Impact: Pilot scope may include non-Krasan roles (with Jim's backing)

### 6. Microsoft Dynamics — Configuration vs Code
- Configuration generation differs from traditional code generation
- Need Dynamics experts (Kashif Ali, Shyam) to understand what's possible
- **Dinkar will investigate Dynamics demo environment** outside the state for experimentation

### 7. Measurement — Tangible Outputs, Not Points
- **Don't measure in story points** — measure in lines of code, documentation generated, configuration lines
- Explicitly allocate training time and adjust KPIs/expectations accordingly
- Clearly communicate methodology and capacity impacts to stakeholders
